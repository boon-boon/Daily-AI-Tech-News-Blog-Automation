# Daily Tech Pulse — Web Frontend

Modern, dark-first, futuristic AI tech news blog. Vanilla HTML/CSS/JS. Zero build step. Deploys to GitHub Pages.

## Folder structure

```
web/
├── index.html             # Homepage
├── article.html           # Per-article template (filled by Python generator)
├── 404.html
├── robots.txt
├── feed.xml               # RSS — generated by Python publisher
├── sitemap.xml            # generated
└── assets/
    ├── css/
    │   ├── main.css       # Design system + components
    │   └── article.css    # Per-article prose styles
    ├── js/
    │   └── main.js        # Nav, scroll reveal, magnetic CTA, newsletter form
    └── img/
```

## Local preview

```bash
cd web
python3 -m http.server 8080
# open http://localhost:8080
```

## Deploy to GitHub Pages

1. Push the contents of `web/` to a repo (or set `web/` as the Pages source folder).
2. In repo **Settings → Pages**, set the branch and folder.
3. Update `BLOG_BASE_URL` in your project `.env` to the Pages URL.
4. The Python generator's `LocalPublisher` writes HTML files; copy them into `web/posts/` (or wherever your routing expects) before each commit. A simple pattern:

```bash
# After running the daily pipeline:
mkdir -p web/posts
cp output/html/*.html web/posts/
git add . && git commit -m "Daily update" && git push
```

You can automate this with a GitHub Actions workflow that runs `main.py --once` on a schedule and commits the output.

## Filling `article.html`

The Python generator emits a `GeneratedArticle` with all the fields below. Replace the `{{TOKENS}}` in `article.html` with your generator's values:

| Token | Source field |
|---|---|
| `{{ARTICLE_TITLE}}` | `article.title` |
| `{{META_DESCRIPTION}}` | `article.meta_description` |
| `{{CANONICAL_URL}}` | `article.canonical_url` |
| `{{OG_IMAGE}}` | first `image_suggestions[].url` (or default) |
| `{{PUBLISHED_ISO}}` | `article.published_at.isoformat()` |
| `{{PUBLISHED_HUMAN}}` | `published_at.strftime('%B %d, %Y')` |
| `{{CATEGORY}}` | `article.category` |
| `{{AUTHOR}}` | `BLOG_AUTHOR` env |
| `{{READING_TIME}}` | `len(body_markdown.split()) // 200` |
| `{{TLDR}}` | `article.tldr` |
| `{{BODY_HTML}}` | `markdown.markdown(article.body_markdown)` |
| `{{FAQ_HTML}}` | rendered as `<details><summary>Q</summary><p>A</p></details>` per FAQ |
| `{{SOURCES_HTML}}` | rendered as `<li><a>label</a></li>` per source |
| `{{TAGS_HTML}}` | rendered as `<span class="chip">tag</span>` per tag |
| `{{JSON_LD_BLOCKS}}` | `article.structured_data.blog_posting` and `faq_page` wrapped in `<script type="application/ld+json">` |

A simple Python helper to populate the template is a good next step (a `WebPublisher` class living next to `LocalPublisher`).

## Customizing

- **Brand colors** — edit `--accent-blue`, `--accent-purple`, `--accent-cyan` in `:root` of `main.css`.
- **Logo** — replace the inline `<svg>` in the `.nav__brand` block of `index.html`.
- **Hero copy** — edit the `<h1>` and `.hero__lede` in `index.html`.
- **Newsletter provider** — wire the `form.addEventListener('submit', ...)` block in `main.js` to your provider's endpoint (Buttondown, Mailchimp, ConvertKit, etc.).

## Performance

- ≤ 30KB CSS, ≤ 5KB JS (gzipped).
- No fonts blocking render — Inter loaded with `preconnect`.
- All animations respect `prefers-reduced-motion`.
- Sticky nav uses `position: sticky` (no JS jank).
- Card hover effects are GPU-friendly (transform + opacity only).

## Accessibility

- Semantic landmarks (`<header>`, `<nav>`, `<main>`, `<footer>`).
- ARIA labels on icon-only buttons and the mobile drawer.
- Visible focus rings (`:focus-visible`).
- Color-contrast tested against WCAG AA on the dark surface.
- `prefers-reduced-motion` shuts down the animated mesh and reveal animations.
